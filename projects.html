KlapTrap Simulator
Role: Game Designer

Klap Trap is a simulator game developed within 24 hours during the Cyberiada contest as part of the Lem Game Jam, where all projects were inspired by Stanisław Lem’s novels. The game is based on “Wielkie Lanie” and follows the story of Trurl, who must escape after fulfilling Klapaucius’ wishes.
As a Game Designer, I was responsible for designing core game mechanics, interaction logic, and screen layouts with a strong focus on user experience. I created UI mockups for the graphics team and prepared detailed specifications of game mechanics and user flows for the programmer, ensuring clear communication between design and development.

Kraska
Role: Game Designer

Kraska is an adventure game project created during the Cyberiada contest. The game is set in an old Polish village several hundred years ago, inspired by Slavic beliefs and folklore. Players take on the role of Kraska, a young girl who seeks the help of a local witch to save her ill mother, while rescuing villagers possessed by demons along the way.
In this project, I was responsible for designing the game script, mechanics, interaction logic, puzzles, and map and object layouts. I conducted extensive research on Slavic culture, beliefs, and customs to ensure historical and cultural authenticity, translating this knowledge into consistent gameplay logic, narrative structure, and visual design.

Serwis Biblioteczny
(student project)

A web-based library management system designed and implemented as an end-to-end project. The application allows users to browse and search books, manage a borrowing cart, and borrow books after logging in. The system includes role-based access for readers, staff members, and administrators.
The application features an administrative panel for managing books, users, categories, tags, and borrowings. Books are organized using hierarchical categories and multiple tags, with advanced search functionality supporting logical operators (AND, OR, NOT). The system also tracks borrowing statuses (available, reserved, borrowed) and provides borrowing history for users.
